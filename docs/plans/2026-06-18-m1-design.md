> 日期:2026-06-18

## 1. 目标

一条命令跑完一局多模型德扑:
```
pokermind match --p1 deepseek:deepseek-v4-flash --p2 glm:glm-4.6 --hands 100
```
- 两个模型(可以是同 provider 不同模型,或不同 provider)Heads-up 打 100 手
- 全过程(每手每动作含内心戏)入 SQLite
- 一局结束更新双方 ELO,打印排行榜
- **此刻项目的差异化价值(内心戏可查)首次落地**

## 2. 关键设计

### 2.1 一局(Match)的语义

- 双方各带起始筹码(默认 1000)。**不重置筹码** —— 每手结束后筹码累积,直到打完 N 手或一方破产。
- button 每手轮换(SB / BB 交替)。
- 一局赢家 = N 手后筹码更多的那方;相等则平局(ELO 不变)。
- 中途破产(某方筹码 < BB)提前结束。

### 2.2 SQLite Schema(4 张表)

```sql
-- 模型注册表:provider:model 作主键,ELO 跨局累积
CREATE TABLE players (
    id            INTEGER PRIMARY KEY,
    provider      TEXT NOT NULL,        -- "deepseek" / "glm"
    model         TEXT NOT NULL,        -- "deepseek-v4-flash" / "glm-4.6"
    label         TEXT NOT NULL,        -- 显示名 "deepseek-v4-flash"
    UNIQUE(provider, model)
);

-- 一局(两个模型打 N 手)
CREATE TABLE games (
    id            INTEGER PRIMARY KEY,
    p1_id         INTEGER NOT NULL REFERENCES players(id),
    p2_id         INTEGER NOT NULL REFERENCES players(id),
    hands_played  INTEGER NOT NULL,
    winner_id     INTEGER,              -- NULL = 平局
    p1_final_chips INTEGER NOT NULL,
    p2_final_chips INTEGER NOT NULL,
    started_at    TEXT NOT NULL,        -- RFC3339
    finished_at   TEXT NOT NULL,
    config_json   TEXT NOT NULL         -- {sb,bb,starting_stack,hands}
);

-- 一手牌(属某局)
CREATE TABLE hands (
    id            INTEGER PRIMARY KEY,
    game_id       INTEGER NOT NULL REFERENCES games(id),
    hand_index    INTEGER NOT NULL,     -- 局内第几手(1-based)
    button_seat   INTEGER NOT NULL,     -- 0 或 1(哪个 seat 是按钮)
    winner_id     INTEGER,              -- player id,NULL = 平局
    folded        INTEGER NOT NULL,     -- 0/1
    pot           INTEGER NOT NULL,
    p1_hole       TEXT NOT NULL,        -- "As Kh"(赢家摊牌或双方都摊时填)
    p2_hole       TEXT NOT NULL,
    community     TEXT NOT NULL         -- "Th 7c 6h"(空街后逐渐增长)
);

-- 一个动作(属某手)。内心戏就存在这里 —— 项目核心差异点。
CREATE TABLE actions (
    id            INTEGER PRIMARY KEY,
    hand_id       INTEGER NOT NULL REFERENCES hands(id),
    seq           INTEGER NOT NULL,     -- 手内序号(0-based)
    street        TEXT NOT NULL,        -- preflop/flop/turn/river
    seat          INTEGER NOT NULL,     -- 0 或 1
    player_id     INTEGER NOT NULL REFERENCES players(id),
    action_type   TEXT NOT NULL,        -- fold/call/raise
    amount        INTEGER NOT NULL,     -- raise-to 总额,其他类型 0
    pot_before    INTEGER NOT NULL,     -- 动作前底池(含本街 bets)
    to_call       INTEGER NOT NULL,     -- 动作时 ToCall
    -- 内心戏(规则 bot 全 NULL)
    reasoning     TEXT,
    hand_strength REAL,
    estimated_equity REAL,
    is_bluffing   INTEGER               -- 0/1/NULL
);
CREATE INDEX idx_actions_hand ON actions(hand_id);
```

> 设计要点:`actions` 表把 LLM 的 SelfReport 摊平成 4 列(reasoning/hs/eq/bluff),而不是塞 JSON blob —— 这样以后能 `SELECT AVG(hand_strength) FROM actions WHERE player_id=X AND action_type='raise'` 直接做分析。这是项目「把内心戏做透」的核心。

### 2.3 ELO

标准 ELO:
- 初始 1500,K 因子 32
- 一局结束按胜负更新:`E_A = 1/(1+10^((R_B-R_A)/400))`,`R_A += K*(S_A - E_A)`,S 是实际得分(赢 1 / 平 0.5 / 输 0)
- ELO 存在 `players` 表的扩展字段里?—— 不,schema 里 `players` 没 elo 列,改在 `players` 加 `elo INTEGER NOT NULL DEFAULT 1500`。

> 修正上面 schema:`players` 表追加 `elo INTEGER NOT NULL DEFAULT 1500`。

### 2.4 Recorder

`internal/store/store.go` 提供:
- `Open(path) (*Store, error)` —— 打开 SQLite,自动 migrate(建表 `IF NOT EXISTS`)
- `RegisterPlayer(provider, model, label) (playerID, error)` —— 幂等
- `GetElo(playerID) int`
- `RecordGame(...) (gameID, error)` —— 一局的事务性写入:games/hands/actions 三表同事务
- `Leaderboard() []LeaderboardRow` —— 按 ELO 降序,附对局数/胜率/累计筹码净值

事务边界:**一局 = 一个事务**(写完 100 手一次性 commit,失败回滚)。这样既快(避免每动作一次 fsync),又保证局内一致性。

### 2.5 Match 编排

`internal/match/match.go`:
```go
type PlayerSpec struct {
    Provider string
    Model    string
    Label    string
}

type Result struct {
    Winner       int          // 0/1/-1(平)
    FinalChips   [2]int
    HandsPlayed  int
}

// Play 跑一局,p1 是 seat0 第一手的 BB(等同 button=1,即 p1 先当 BB,p2 先当 SB)。
// 或者约定 p1 先当 button —— 简单点,p1 先当 button(seat0),每手轮换。
// 每手调 engine.PlayHand,把事件流喂给 recorder 落库。
// 当筹码 < bigBlind 时提前结束。
func Play(p1, p2 PlayerSpec, hands int, cfg engine.Config, rec *store.Store) (*Result, error)
```

编排:
1. 构造两个 `LLMPlayer`(按 provider+model),包成 `engine.PlayerSeat`
2. 注册两个 player 到 store(拿 playerID)
3. 循环 N 手:
   - 各自 stack = 上一手结束的 stack(不重置)
   - 调 `engine.PlayHand`
   - 把 events 翻译成 `actions/hands` 落库(累积在内存,事务一次性 commit)
   - button 翻转
   - 若某方 stack < BB,提前结束
4. 一局结束:写 `games` 行,更新两方 ELO
5. 返回 Result

> 注:引擎 `PlayHand` 当前每手独立用 seats.Stack。Match 层要在循环外维护双方 stack,每手把当前 stack 注入新 PlayerSeat。引擎不感知「跨手累积」,这是 Match 的职责。

### 2.6 CLI

`pokermind match --p1 <provider:model> --p2 <provider:model> --hands N [--db path]`

- `--p1/--p2` 格式 `provider:model`(如 `deepseek:deepseek-v4-flash`)
- `--hands` 默认 100
- `--db` 默认 `pokermind.db`(自动建)
- 跑完打印:每手结果简表 + 最终 ELO 排行榜(从 DB 查所有 player)
- `--verbose` 打印每个 LLM 动作的 reasoning(默认只打印每手赢家 + pot)

## 3. 单测覆盖

- **elo**:纯函数,赢/输/平各一个 case,断言 rating 变化符合公式
- **store**:用临时文件建 DB,RegisterPlayer 幂等、RecordGame 事务、Leaderboard 排序
